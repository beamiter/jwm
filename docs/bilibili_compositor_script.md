# JWM X11 Compositor 功能介绍 - Bilibili 视频脚本

## 开场白（约30秒）

大家好，欢迎来到本期视频。今天我要给大家介绍的是 JWM 窗口管理器的 X11 合成器。

JWM 的合成器完全基于 OpenGL 3.3 Core，用 Rust 编写，使用 glow 库做 GL 绑定。它实现了从基础的阴影、圆角、透明度，到高级的果冻窗口、3D 工作区切换、精灵最小化等一整套视觉效果。

接下来我会一个一个给大家演示这些功能。

---

## 第一部分：阴影（Shadow）

好，我们先来看最基础的——窗口阴影。

可以看到，每个窗口后面都有一层柔和的阴影。这个阴影不是简单的贴图，而是用 **SDF（有符号距离场）** 在 shader 里实时计算的。它的原理是：在窗口边界外扩展一圈 quad，然后用 SDF 计算每个像素到圆角矩形边缘的距离，再通过 smoothstep 做衰减，得到一个非常平滑的阴影效果。最后对 alpha 做了平方处理，让阴影看起来更柔和自然。

阴影有这么几个可配置参数：
- `shadow_radius`：阴影的扩散半径，默认 24 像素
- `shadow_offset`：阴影偏移，默认向右下各偏 4 像素
- `shadow_color`：阴影颜色和透明度
- `shadow_bottom_extra`：底部额外偏移，让底部阴影更重，模拟光从上方照射的效果
- `shadow_exclude`：可以按窗口类名排除，比如 dock 就不需要阴影

我把阴影关掉给你们看看对比。关掉之后窗口就像贴在桌面上一样，没有层次感了。再打开——立体感就回来了。

## 第二部分：圆角（Rounded Corners）

接下来看圆角。

JWM 的圆角同样是用 SDF 在 fragment shader 里实现的。`rounded_rect_sdf` 函数计算每个像素到圆角矩形边界的距离，然后用 `smoothstep(-1.0, 1.0, dist)` 做反锯齿。注意这里有个细节：不仅 alpha 通道被裁切了，RGB 通道也同时被 mask 掉了。这是因为我们用的是**预乘 alpha 混合**，如果只裁 alpha 不裁 RGB，角落上会出现白边或颜色溢出。

圆角默认半径是 10 像素。可以通过 `corner_radius_rules` 按窗口类名单独设置，比如给终端设大一点的圆角，给某些窗口设成 0 变成直角。`rounded_corners_exclude` 可以完全排除某些窗口的圆角处理。

注意圆角和阴影是配合工作的——阴影 shader 也使用了同样的 corner radius，所以阴影的形状和窗口完全吻合。

---

## 第三部分：透明度与模糊（Opacity & Blur）

现在我们来看透明度和背景模糊。

JWM 支持对非活动窗口自动降低透明度，默认 `inactive_opacity` 是 0.98，就是稍微透一点点。你也可以通过 `opacity_rules` 按窗口类名精确控制，比如 `"90:Alacritty"` 让终端固定 90% 不透明。

更重要的是**背景模糊**。当窗口有透明度的时候，如果不做模糊，透过去看到的内容会很杂乱。JWM 用的是 **Dual Kawase Blur** 算法。

它的工作原理是这样的：
1. 先把当前合成好的场景拷贝到一个 FBO
2. 做降采样（downsampling）：每一级分辨率减半，采样时取四个对角像素做平均
3. 到底之后再做升采样（upsampling）：每一级分辨率翻倍，取八个邻近像素混合
4. 这样来回一趟就得到了高质量的模糊效果

`blur_strength` 控制 mipmap 级数，1 到 5，数字越大越模糊。默认是 3 级。

还有一个 **frosted glass（毛玻璃）** 功能，可以按规则让特定窗口拥有磨砂玻璃效果。还有**模糊缓存**——如果只是毛玻璃窗口本身在变化，背景没变，就直接复用上一帧的模糊结果，避免重复计算。

## 第四部分：窗口动画（Window Animations）

### 4.1 开窗/关窗动画

窗口打开时有一个从缩小到正常大小的缩放动画，同时配合淡入。关闭时反过来，缩小加淡出。缩放起点由 `window_animation_scale` 控制，默认 0.85，就是从 85% 大小开始放大到 100%。

### 4.2 窗口打开水波纹（Ripple on Open）

这是一个比较酷炫的效果。窗口打开的瞬间，会产生一个从中心向外扩散的水波纹。

实现原理是在 fragment shader 里对 UV 坐标做径向扰动：以窗口中心为原点，计算每个像素的距离，然后用 sin 函数产生波形，只在波前附近渲染，并且随时间衰减。看起来就像在水面上投了一颗石子。

### 4.3 果冻窗口（Wobbly Windows）

这个是经典的 Compiz 同款效果。拖动窗口时，窗口会像果冻一样晃动。

实现上，每个窗口的表面被分成一个 N×N 的弹簧质点网格，默认 8×8。每个网格节点都有位置和速度，节点之间用弹簧连接。拖动时，鼠标拖着的那个锚点移动了，但其他节点因为弹簧力要过一会儿才跟上来，所以就产生了弹性形变。

物理模拟用的是 **Symplectic Euler 积分**，每帧分 3 个子步计算，保证数值稳定。弹簧的刚度、阻尼都可以调。`wobbly_stiffness` 越大窗口越硬，`wobbly_damping` 越大晃动衰减越快。

在 vertex shader 里，这些网格偏移量通过 uniform 数组传进去，对顶点坐标做变形，渲染出来的窗口就是弯曲的了。

### 4.4 精灵最小化（Genie Minimize）

模仿 macOS 的经典效果。最小化窗口时，窗口不是直接消失，而是像被吸进 dock 一样变形收缩。

在 vertex shader 里，根据动画进度 t，窗口的底部行先向 dock 位置收缩，宽度缩到 2%，顶部行跟着过去但速度更慢，整体形成一个漏斗状的变形，非常有趣。动画时长可以通过 `genie_duration_ms` 配置，默认 300 毫秒。

### 4.5 拖动残影（Motion Trail）

拖动窗口时，窗口身后会留下几个半透明的残影。实现很简单：维护一个环形缓冲区记录历史位置，渲染时在这些历史位置画半透明的窗口副本。默认 5 帧残影，透明度 0.3。

### 4.6 关窗粒子特效（Particle Effects）

关闭窗口时，窗口碎裂成一堆粒子向四周飞散。

每个粒子有位置、速度、剩余生命值。初始速度向上偏置，之后受重力下落。颜色根据粒子在网格中的位置生成渐变。粒子在 shader 里画成带柔和边缘的圆形。默认 150 个粒子，生命周期 0.8 秒。

## 第五部分：工作区切换动画（Workspace Transitions）

切换工作区是一个窗口管理器非常高频的操作。JWM 实现了十种切换动画：

1. **Slide（滑动）**：默认效果，新旧工作区水平滑入滑出
2. **Cube（立方体）**：经典 Compiz 3D 立方体旋转
3. **Fade（淡入淡出）**：旧工作区淡出，新工作区淡入
4. **Flip（翻转）**：像翻书一样绕中轴 Y 旋转
5. **Zoom（缩放）**：旧工作区缩小消失，新的放大出现
6. **Stack（堆叠）**：新工作区从上方落下覆盖旧的
7. **Blinds（百叶窗）**：像百叶窗一样一条条翻转切换
8. **Coverflow**：类似 iTunes 的专辑封面流效果
9. **Helix（螺旋）**：双螺旋旋转过渡
10. **Portal（传送门）**：旧工作区从中心向外碎裂，新的从中心展开

这些动画的实现原理是：在切换前，把当前场景渲染到一个 FBO 保存下来。然后切换 tag，把新场景也渲染到另一个 FBO。最后在过渡期间，根据不同动画模式对这两张"截图"做 3D 变换和混合。

过渡动画使用 Ease-Out 三次缓动函数，让动画有一个自然的减速过程。时长可以通过 `transition_duration` 配置。

## 第六部分：3D 窗口倾斜（Window Tilt）

这也是一个非常有视觉冲击力的效果。开启后，鼠标悬停在窗口上时，窗口会根据鼠标位置做轻微的 3D 倾斜。

实现用的是完整的 3D 变换：
1. 根据鼠标相对于窗口中心的偏移，计算目标旋转角度
2. 在 vertex shader 里构建旋转矩阵（先绕 X 轴再绕 Y 轴）
3. 应用透视投影，`tilt_perspective` 控制观察者距离，越小透视越强
4. 叠加 Blinn-Phong 高光，让倾斜的表面有反光效果
5. 边缘做类 Fresnel 变暗，增强立体感

角度变化通过插值平滑过渡，不会突变。`tilt_amount` 控制最大倾斜弧度，默认 0.26 弧度，大约 15 度。

## 第七部分：特殊模式

### 7.1 Alt-Tab 概览（Overview）

按 Alt-Tab 时进入概览模式。JWM 的概览不是普通的平面列表，而是一个 **3D 六棱柱** 布局。窗口缩略图贴在棱柱的各个面上，切换时棱柱旋转到下一个面。每个缩略图下面有窗口标题。背景有暗色遮罩和暗角效果。

### 7.2 Expose / Mission Control

类似 macOS 的 Mission Control。激活后所有窗口自动排列成网格，保持宽高比，可以直接点击切换。有平滑的展开/收起动画和鼠标悬停高亮。

### 7.3 窗口吸附预览（Snap Preview）

拖动窗口到屏幕边缘时，会显示一个半透明的蓝色矩形预示吸附后的位置和大小，松手就吸附到那个位置。这对平铺操作非常直观。

### 7.4 老板键（Window Peek）

一键隐藏所有窗口，带淡出动画。再按一下全部恢复。可以通过 `peek_exclude` 排除某些窗口不被隐藏。

### 7.5 窗口标签页（Window Tabs）

可以把多个窗口合并成标签页的形式，顶部显示标签栏，在同一个窗口框架内切换不同的应用。

## 第八部分：后处理效果（Post-Processing）

合成完成后，JWM 还有一套完整的后处理管线：

- **色温调节**：模拟暖光/冷光，配合 **Night Light（夜灯模式）** 可以定时自动开启
- **饱和度 / 亮度 / 对比度**：全局调节
- **反色模式**：颜色翻转
- **灰度模式**：去除所有颜色
- **色盲校正**：支持红色盲、绿色盲、蓝黄色盲三种模式，用 Hunt-Pointer-Estevez LMS 色彩空间变换实现

还有一个**放大镜**功能，跟随鼠标的圆形放大区域，可以调节半径和倍率。

---

## 第九部分：辅助功能

### 9.1 屏幕标注（Annotations）

可以直接在屏幕上用画笔标注，用 GL_LINES 实时渲染。颜色和线宽都可以配置。这个功能在做演示或者录屏讲解时特别有用。

### 9.2 聚焦高亮（Focus Highlight）

窗口获得焦点时，边框闪烁高亮同时窗口做一个 1.02 倍的弹跳缩放，让你一眼看到焦点切换到了哪里。

### 9.3 Attention / Urgent 动画

当某个窗口发出紧急提醒（比如消息通知）时，窗口边缘会以橙色脉冲闪烁提示。

### 9.4 屏幕边缘发光（Edge Glow）

鼠标靠近屏幕边缘时，最近的边缘发出柔和的蓝色光晕。这是一个纯视觉效果，让操作有种赛博感。

## 第十部分：性能优化

最后聊一下性能优化，毕竟特效做得再好，卡顿了也白搭。

JWM 的合成器在性能方面做了几个关键优化：

1. **Tile-Based Damage Tracking（分块脏区追踪）**：屏幕被分成 8×6 的瓦片网格，只重绘有变化的瓦片区域，用 GL 的 scissor test 裁切
2. **模糊缓存**：背景没变化时跳过昂贵的模糊计算
3. **GPU Fence 异步刷新**：用 GL sync 对象做异步 pixmap 更新，避免 CPU 等待 GPU
4. **全屏直通（Fullscreen Unredirect）**：全屏窗口自动绕过合成器直接输出，零开销
5. **动画期间自动降低模糊质量**：检测到工作区切换等动画时自动降低 blur level
6. **场景不变时跳过渲染**：如果没有任何窗口变化也没有动画在播放，直接跳过整个 render_frame

这些优化保证了日常使用时 GPU 负载很低，只在有动画时才会有额外开销。

好，以上就是 JWM X11 合成器的全部功能介绍。从基础的阴影圆角，到果冻窗口、3D 工作区切换、粒子特效，再到后处理和无障碍功能，全部用 OpenGL 3.3 + Rust 实现，代码完全自研，不依赖任何外部合成器库。
